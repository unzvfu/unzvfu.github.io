I would like you to write a website for me. At a high level, the goal of this website is to allow a group of people to collaboratively choose a name for our company through voting. The website should be simple and straight forward; for example, similar to the Doodle scheduling website. The full description is as follows.

- The website has users. A user is just given by their username which they enter on first connection. There is no authentication or anything, this is based on trust and is not public facing.

- The website maintains a list of candidate company names. New names can be added by users. The user can add as many names as they like. There is a way to associate text notes and URLs, and there should be a way for users to add comments (which identify the commenter by username) to each candidate name. Any user can modify the data associated to a candidate name.

- Each user has a page that only they can modify where they can rank the currently registered candidate names in order from their most favourite to their least favourite. This ordering is specific to the user and only the user with the associated username can change the ordering. They are not obliged to rank every name; any unranked names are automatically considered to have the lowest rank.

- When a new name is added by a user, they must rank it immediately. For other users, this new name is initially added to a separate list of their "currently unranked" names which they can rank at their leisure. From the system's point of view, unranked names can be considered to be ranked at the lowest level.

- There should be a special page that any user can view that dynamically provides the current overall ranking of the names. The page should provide the ranking of all the names according to several different voting systems. The first and most prominent system should be by "instant run-off voting" followed by "first past the post"

The website must be easy to deploy; I want to be able to deploy it on my github.io account. It should only be discoverable and visit-able by people who are given a "secret link". Provide instructions on how to do all this.
